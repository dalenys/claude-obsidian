# Full Audit Remediation Design

## Goal

Correct all twelve confirmed audit defects and harden the unpinned Excalidraw
download without changing the plugin's public workflows or adding dependencies.

## Constraints

- Preserve macOS Bash 3.2 and Linux compatibility.
- Preserve filesystem fallback behavior and existing command-line interfaces.
- Keep runtime state writes atomic.
- Default to no network egress and no destructive overwrite.
- Add hermetic regression coverage for every executable behavior change.
- Do not perform unrelated lint cleanup or refactoring.

## Design

### Retrieval integrity

`contextual-prefix.py` will reconcile generated chunk directories with current
page content. Reprocessing a page removes surplus chunks after successful
writes, empty pages remove their former chunks, and `--all` removes directories
whose source pages no longer exist. Cleanup must not run in `--peek` mode.

Rerank fallback will preserve `bm25_score`. Retrieval will rank all BM25
candidates before page-level deduplication, then apply `--top`, so duplicate
chunks cannot reduce the requested number of distinct pages.

### Lock lifecycle and hooks

Wiki locks are cross-process leases. Stale cleanup will therefore use age only,
not PID liveness. A fresh lock must survive SessionStart cleanup.

Lock-protected workflows will explicitly trigger the repository's staging and
auto-commit path after the final release, because Bash release commands do not
produce a `PostToolUse` event. Documentation must not promise a commit that
cannot occur.

### Installer and transport safety

DragonScale setup will rebuild the address counter through the allocator when
the counter is missing. Vault setup will preserve existing Obsidian JSON files
unless the user explicitly requests replacement; forced replacement creates
backups.

Transport detection will capability-test supported Obsidian CLI commands rather
than accepting a binary by name. Snapshot JSON will be generated by Python's
JSON encoder. Retrieval setup will parse and retain
`--allow-remote-ollama`.

The Excalidraw download will use a pinned release artifact, fail on HTTP errors,
verify a committed SHA-256 digest, and atomically rename the verified file.

### Routing and skill contracts

Every methodology-mode route must normalize under `wiki/` and reject absolute
or escaping configured folders. Claude Code skill allowlists will include Bash
where required. Ingest hashing guidance will feature-detect portable SHA-256
commands, preferring `shasum -a 256` on macOS.

## Verification

Each defect receives a regression test that fails against the baseline. Final
verification consists of targeted regression suites, `make test`, ShellCheck,
Ruff, JSON/frontmatter validation, and inspection of the complete branch diff.
